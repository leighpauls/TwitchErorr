TwitchErorr is a quick and easy way to test how your HTTP proxy or client handles different HTTP status codes.


--Usage--
With either a GET or POST request, supply the following arguments:

error (required)
Any HTTP status code you can think of. Invalid or crazy codes may get transformed into something more "reasonable" by the server, but any code you can expect out in the wild should work fine.
If not provided, then this splash page will be displaed.

delay (optional)
The delay in milliseconds that the server should wait before sending the response (including the header).
If not provided, then the reposnse will be sent immediately.

content (optional)
Content to be placed into the body of the reposnse.
If not provided, then the empty string "" will be used.


--Example--
To return a 506 error, with a 5.3 second delay, containing the string "Eat my shorts", use:

http://twitcherorr.nodester.com/?error=506&delay=5300&content=Eat%20my%20shorts
